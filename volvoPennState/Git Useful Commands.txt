Why is .git/lfs/objects so big?
That folder stores all Git LFS-tracked files — even old versions. By default, Git LFS doesn’t delete old blobs, so over time it grows and eats your disk space.

✅ Step-by-step to reclaim space
🧼 1. Prune old LFS objects (the safe way)
This removes unreferenced old LFS files:


git lfs prune


This is the official Git LFS way to remove local objects that aren’t needed anymore.

2. Check what’s taking up space
Want to see what’s big before deleting?


du -sh .git/lfs/objects/*
